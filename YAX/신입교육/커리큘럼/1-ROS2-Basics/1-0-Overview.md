## 1. Phase 1 소개

### 학습 목표
1. ROS2의 핵심 개념(Node, Topic, Service, Action)을 이해할 수 있다
2. ROS2 CLI 도구를 능숙하게 사용할 수 있다
3. Python으로 ROS2 노드를 작성할 수 있다
4. Launch 파일을 작성하여 여러 노드를 실행할 수 있다
5. RQT 도구로 시스템을 모니터링할 수 있다

### 전체 소요 시간
- 오프라인 세션: 8시간 (4주 × 2시간)
- 자율학습: 8시간
- **총 16시간**

### 왜 ROS2인가?
ROS2는 로봇 소프트웨어 개발의 표준 프레임워크입니다. F1TENTH 차량의 모든 센서와 제어기를 연결하고, 자율주행 알고리즘을 구현하는 데 ROS2를 사용합니다.

## 2. 핵심 개념 미리보기

### 2.1 ROS2 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                        ROS2 시스템                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [LiDAR 노드]──/scan──►[알고리즘 노드]──/cmd_vel──►[VESC 노드]  │
│        │                    │                     │         │
│        └────────────────────┴─────────────────────┘         │
│                         DDS                                 │
│                  (통신 미들웨어)                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 통신 패턴

| 패턴 | 용도 | 예시 |
|------|------|------|
| **Topic** | 연속적인 데이터 스트림 | 센서 데이터, 제어 명령 |
| **Service** | 일회성 요청-응답 | 파라미터 설정, 상태 조회 |
| **Action** | 장시간 비동기 작업 | 경로 계획, 목표점 이동 |

### 2.3 F1TENTH에서 자주 사용하는 토픽

| 토픽 | 메시지 타입 | 설명 |
|------|-------------|------|
| `/scan` | sensor_msgs/LaserScan | LiDAR 데이터 |
| `/odom` | nav_msgs/Odometry | 오도메트리 |
| `/cmd_vel` | geometry_msgs/Twist | 속도 명령 |
| `/drive` | ackermann_msgs/AckermannDriveStamped | Ackermann 명령 |

## 5. 실습 환경 준비

### 5.1 필수 패키지 설치

```bash
# ROS2 기본 패키지
sudo apt install -y \
    ros-humble-turtlesim \
    ros-humble-rqt \
    ros-humble-rqt-common-plugins

# 개발 도구
sudo apt install -y \
    python3-colcon-common-extensions \
    python3-rosdep
```

### 5.2 워크스페이스 준비

```bash
cd ~/f1tenth_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
```

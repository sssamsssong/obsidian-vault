# Phase 4: 자율주행 알고리즘 개요

> YAX F1TENTH 교육자료 | Phase 4: Autonomy
> 
> 기간: 15-18주차

---

## 1. Phase 4 소개

### 학습 목표
1. 반응형 장애물 회피 알고리즘을 이해하고 구현할 수 있다
2. 경로 추종 알고리즘을 이해하고 구현할 수 있다
3. 시뮬레이션과 실차에서 알고리즘을 테스트할 수 있다
4. 파라미터 튜닝을 통해 성능을 최적화할 수 있다

### 전체 소요 시간
- 오프라인 세션: 8시간 (4주 × 2시간)
- 자율학습: 12시간
- **총 20시간**

### 왜 알고리즘인가?
센서와 제어가 준비되면 이제 "어떻게 주행할 것인가"를 결정해야 합니다. 이것이 자율주행 알고리즘의 영역입니다.

---

## 2. 알고리즘 분류

### 2.1 반응형 (Reactive)

지도 없이 센서 데이터만으로 즉각적으로 반응합니다.

| 알고리즘 | 특징 | 장점 | 단점 |
|---------|------|------|------|
| Wall Following | 벽과 일정 거리 유지 | 단순, 안정적 | 최적 경로 아님 |
| Follow the Gap | 가장 큰 빈 공간 추종 | 동적 장애물 대응 | 진동 가능 |

### 2.2 계획 기반 (Planning-based)

지도와 경로를 사용하여 최적 경로를 추종합니다.

| 알고리즘 | 특징 | 장점 | 단점 |
|---------|------|------|------|
| Pure Pursuit | Lookahead 포인트 추종 | 부드러운 경로 | 코너에서 컷 |
| Stanley | 횡방향 오차 최소화 | 정밀한 추종 | 고속에서 불안정 |
| MPC | 모델 예측 최적화 | 최적 제어 | 계산 비용 높음 |

### 2.3 알고리즘 선택 가이드

```
┌─────────────────────────────────────────────────────────────┐
│                     상황에 따른 선택                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  지도가 없다 ──────────► 반응형 알고리즘                     │
│      │                    └── Wall Following               │
│      │                    └── Follow the Gap               │
│      │                                                      │
│  지도가 있다 ──────────► 경로 추종 알고리즘                  │
│      │                    └── Pure Pursuit (고속)          │
│      │                    └── Stanley (저속, 정밀)         │
│      │                    └── MPC (최적화 필요시)          │
│                                                             │
│  동적 환경 ───────────► 반응형 + 경로 추종 혼합             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 5. 개발 워크플로우

### 5.1 시뮬레이션 → 실차

```
┌─────────────────────────────────────────────────────────────┐
│  1. MORAI 시뮬레이션                                        │
│     └── 알고리즘 구현 및 기본 테스트                         │
│     └── 파라미터 초기 튜닝                                  │
│                                                             │
│  2. 실차 저속 테스트                                        │
│     └── 0.5 m/s로 기본 동작 확인                            │
│     └── 안전 기능 확인                                      │
│                                                             │
│  3. 점진적 속도 증가                                        │
│     └── 1.0 → 1.5 → 2.0 m/s                                │
│     └── 각 속도에서 안정성 확인                              │
│                                                             │
│  4. 최적화                                                  │
│     └── 파라미터 미세 조정                                  │
│     └── 랩타임 기록                                         │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 파라미터 튜닝 접근법

1. **보수적으로 시작**: 낮은 속도, 큰 안전 마진
2. **한 번에 하나씩**: 파라미터 하나만 변경
3. **기록 유지**: 각 설정과 결과 기록
4. **롤백 준비**: 이전 설정으로 복귀 가능하게
